# NexDR 修改版 - 快速使用指南

## 📦 一、环境准备

### 1. 进入项目目录

```bash
cd /Users/flynn/Code/agent-beginner/deep-researcher/NexDR
```

### 2. 激活虚拟环境

```bash
# 项目已包含虚拟环境，直接激活
source .venv/bin/activate
```

### 3. 配置环境变量

```bash
# 复制示例文件
cp .env.example .env

# 编辑 .env 文件，填入你的 API Key
vim .env  # 或使用你喜欢的编辑器
```

**必需的环境变量：**

```bash
# LLM 配置（必需）
OPENAI_API_KEY=sk-xxxxx
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o

# Serper API（用于网络搜索，必需）
SERPER_API_KEY=your_serper_api_key

# Semantic Scholar API（可选，免费）
# 不配置也可使用，但有限额（100 次/天）
SEMANTIC_SCHOLAR_API_KEY=your_semantic_scholar_api_key

# 多模态 LLM（用于图片描述，可选）
MULTI_MODAL_LLM_API_KEY=sk-xxxxx
MULTI_MODAL_LLM_BASE_URL=https://api.openai.com/v1
MULTI_MODAL_LLM_MODEL=gpt-4-vision-preview
```

**获取 API Key：**
- Serper API: https://serper.dev/ (免费 100 次/月)
- Semantic Scholar: https://www.semanticscholar.org/product/api (免费 100 次/天)
- OpenAI: https://platform.openai.com/api-keys

### 4. 安装额外依赖（可选）

```bash
# PDF 解析支持
pip install pypdf

# 图片 OCR 支持
pip install pytesseract Pillow

# macOS 安装 tesseract
brew install tesseract
```

## 🚀 二、基本使用

### 场景 1：简单的研究查询

```bash
python quick_start.py \
    --query="人工智能在医疗领域的应用" \
    --report_format=markdown \
    --output_dir=workspaces/research1
```

**输出：**
```
workspaces/research1/
├── markdown_report.md      # 研究报告
├── citations.json          # 引用来源
├── final_state.json        # 运行状态
└── logs_*.log             # 运行日志
```

---

### 场景 2：使用 Semantic Scholar 学术搜索

Agent 会自动根据查询内容选择合适的搜索源。你也可以在系统提示中引导它使用 Semantic Scholar。

```bash
python quick_start.py \
    --query="Transformer 模型的最新研究进展，请使用学术搜索" \
    --report_format=markdown \
    --output_dir=workspaces/research2
```

### 场景 3：启用用户反馈循环（推荐）

```bash
python quick_start.py \
    --query="深度学习在金融风控中的应用" \
    --report_format=markdown \
    --use_feedback_loop \
    --max_iterations=3 \
    --output_dir=workspaces/research3
```

### 场景 4：处理 PDF、图片等多格式输入

```bash
python quick_start.py \
    --query="基于这些文献撰写综述" \
    --report_format=markdown \
    --input_files paper1.pdf paper2.pdf chart.png notes.md \
    --output_dir=workspaces/research4
```

**支持的文件格式：**
- **文本**: .txt, .md, .py, .json, .csv, .yaml, .html
- **PDF**: .pdf（需要 `pypdf` 库）
- **图片**: .jpg, .png, .gif, .bmp, .webp（需要多模态 LLM 或 pytesseract）

**处理过程：**
```
Processing 3 input file(s)...
Successfully processed file: paper1.pdf (45230 chars)
Successfully processed file: paper2.pdf (38921 chars)
Successfully processed file: chart.png (1250 chars)
Processed 3 file(s) successfully
```

**输出：**
```
workspaces/research4/
├── markdown_report.md
├── paper1.pdf.extracted.txt    # PDF 提取的文本
├── paper2.pdf.extracted.txt
├── chart.png.extracted.txt     # 图片描述/OCR 结果
└── ...
```

## 📊 三、输出格式说明

### Markdown 报告

```bash
--report_format=markdown
```
生成 `markdown_report.md`，包含：
- 结构化的研究报告
- 内联引用标注（如【1†S2】）
- 表格和图片（如果有）

### HTML 报告

```bash
--report_format=html
```
生成 `html_report.html`，包含：
- 可导航的幻灯片
- 交互式界面

### Markdown + HTML

```bash
--report_format=markdown+html
```
同时生成两种格式

## ❓ 五、常见问题

### Q1: 提示 "SERPER_API_KEY environment variable is required"

**解决：** 在 `.env` 文件中配置 Serper API Key，或改用 Semantic Scholar。

---

### Q2: PDF 解析失败

**解决：**
```bash
# 安装 pypdf
pip install pypdf

# 如果是扫描版 PDF，需要 OCR
pip install pdf2image pytesseract
brew install tesseract poppler  # macOS
```

---

### Q3: 图片解析失败

**解决：**
1. 配置多模态 LLM（推荐）：
```bash
MULTI_MODAL_LLM_API_KEY=sk-xxx
MULTI_MODAL_LLM_BASE_URL=https://api.openai.com/v1
MULTI_MODAL_LLM_MODEL=gpt-4-vision-preview
```

2. 或使用 OCR（免费但效果有限）：
```bash
pip install pytesseract Pillow
brew install tesseract  # macOS
```

---

### Q4: 反馈循环中如何停止？

**回答：** 在提示 "Press Enter after you've finished editing" 时，输入 `done` 即可停止。

---

### Q5: 如何查看 Agent 的搜索历史？

**回答：** 查看 `final_state.json` 文件，包含完整的执行历史和状态。
